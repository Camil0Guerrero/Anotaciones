Unidades de medida en CSS

En CSS podemos aplicar tamaños a todos los elementos y contenido por aparte. Para ello disponemos de distintas maneras las cuales están agrupadas y enfocadas para distintas tareas. A continuación veremos medidas que nos pueden ayudar a tener varias opciones para elegir la que mejor se acomode a nuestra necesidad

Absolutas

Su valor no cambia, son unidades del mundo real. Estas unidades son utilizadas sobretodo cuando vamos a imprimir una interfaz visual, para que salga tal cual estamos maquetando

  • pc
  • cm
  • mm
  • Q

    • Las mas usadas y útiles para nosotros son:

  • pt (1/72in) Esta medida es usado por los procesadores de texto, word, google docs, etc...
    • in -> pulgada
  • px (1/96)

Relativas

Su valor es dinámico, cambia segun su contexto

Al tamaño de la fuente

  • em -> Basada en la anchura de la "m", de la fuente del elemento
  • rem -> Basada en la anchura de la "", de la fuente del elemento raíz (HTML)
  • ex -> Basada en la altura de la "X" de la fuente del elemento
  • ch -> Basada en la anchura de la "O" del fuente del elemento

Porcentaje

% -> a partir del tamaño del contenedor

Viewport

El area visible

  • vw -> ancho del viewport. De 1 a 100
  • vh -> alto del viewport. De 1 a 100
  • vmax -> Entre vw y vh toma el que tenga mayor valor
  • vmin -> Entre vw y vh toma el que tenga menor valor
El navegador renderiza las medidas en px

16 Es el tamaño por defecto de la fuente

Pixeles

Es una unidad física, una pulgada son 0.96 pixeles

En la medida de lo posible debemos no usar las medidas absolutas. Tratar de siempre usar relativas para que nuestro sitio se vea siempre bien, no solo en algunos casos 

        
          background-color: cadetblue;
          width: 500px;
          height: 400px;
          font-size: 20px;
          border: thick solid rebeccapurple;

Ems

Estas medidas van acorde a los pixeles que nuestro font size tiene

En el caso ejemplo tomara como referencia el ancho de la m pero con 24px que es el que tiene su padre definido en el font-size

Al usar esta medida debemos tener en cuenta cual es el valor que tenemos como referencia

        
          font-size: 24px;
                    --24px   12px
          padding: 1em 0.5em; 
        
      
          
              font-size: 2em; 48px
              margin: 0.5em; 24px
              padding: 1em; 48px

Rems

Root em, es parecido solo que este siempre toma como referencia el tamaño de la fuente pero de nuestro elemento HTML

Por lo tanto no importa si hemos definido en elementos valores distintos en el tamaño de la letra

        
          font-size: 24px;
          padding: 1rem 0.5rem; 16px 8px
        
      
          
            background-color: salmon;
            border: thick solid beige;
            font-size: 2rem; 32px por defecto
            margin: 0.5rem;8px
            padding: 1rem; 16px

Exs

La X es la letra mas alta, por ello se toma como referencia, esta propiedad no es muy usada. Ya que casi siempre nos preocupamos por el ancho y nuestro alto se va acomodando a partir de este. Esta propiedad tal vez nos serviría cuando queramos poner nuestro sitio en horizontal

Como es tan poco usada esta medida, no existe "Rex"

        
          font-size: 24px;
          padding: 1ex 0.5ex;
        
      
          
            font-size: 2ex;
            margin: 0.5ex;
            padding: 1ex;

Chs

Toma como referencia el ancho de la O. La mejor funcionalidad es cuando tenemos el conocimiento de cuantos caracteres tiene nuestro elemento y queremos que el ancho quede perfectamente

Toma como referencia el tamaño de la fuente del elemento como em

1234567890

Porcentajes

Esta unidad de medida va de acuerdo al tamaño del elemento padre

Cuando queremos definir la altura es distinto, ya que la mayoría de las veces la altura del elemento va a estar delimitado por el contenido.
Lo recomendado es no definir la altura, que se haga a partir de nuestro contenido, asi si hacemos modificaciones en nuestro elemento HTML no debemos preocuparnos por los estilos

Dentro de un elemento que tenga definida la altura y anchura tal vez si podríamos usarlo

        
          width: 80%;
          padding: 5%;
          margin: 20% 10%;

Viewport

Con esta unidad si podemos hacer que un elemento ocupe toda la pantalla. Este no tiene problema como los porcentajes. Muy funcional para por ejemplo las hero-image

Esta definido por el tamaño de la pantalla

Para definir el ancho no es muy recomendado ya que este tiene en cuenta el margen del body y ademas, nuestra barra de navegación, el scroll, también hace parte del viewport. Por lo que podríamos tener problemas

Esta unidad permite fluir, que nuestro contenido se vaya adaptando al entorno

        
          font-size: 5vw;
          padding: 2vh 2vw;

vmax y vmin

Estas unidades casi no son usadas ya que puede ser algo complejo hacer que siempre se vea bien nuestro elemento 

        
            width: 50vmax;
            height: 50vmin;

¿Cuando usar cada una de las unidades de medida?

Absolutas

  • cm, mm, in -> cuando queremos imprimir nuestro contenido, para que las medidas sean exactas y quede igual en el papel
  • px -> - Para la impresión - Definir el font-size de nuestro HTML

Relativas

  • rem -> Todo lo relacionado con la tipografía: font-size, padding margin
  • em -> Para cuando queremos que el elemento tenga distancias definidas, por ejemplo: pequeño, medio, largo, etc...
  • ex -> Para cuando sabemos cuantas filas de caracteres vamos a recibir, aunque casi no se usa ya que la altura no deberíamos de definiría a menos que sean tarjetas y queremos que sea uniforme
  • ch -> Cuando sabemos los caracteres que va a tener nuestro elemento y queremos limitarlo

Porcentaje

Diseño responsivo

Viewport

Para cuando necesitamos el tamaño de la pantalla como referencia, por ejemplo para el alto de la hero-image, para textos fluidos, etc...
