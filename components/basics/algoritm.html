Algoritmo de CSS

La forma en que el navegador aplica los estilos al documento HTML.

La cascada

Es el mecanismo que tiene el navegador para aplicar los estilos

  1. origen del código

      User Agent Son los estilos que cada navegador aplica por defecto a nuestras etiquetas

    1. personalizaciones

      Las preferencias, temas, estilos que le hemos aplicado a nuestro navegador y computador

    2. Estilos del autor

      Los estilos que ha aplicado el creador de la pagina

  2. Especificidad del selector
  3. Orden de aparición

La especificidad

El peso que tiene un selector
La especificidad se calcula de la siguiente forma:
  1. Etiquetas y PseudoElementos -> 0, 0, 0, 1
  2. Clases, atributos, y PseudoClases -> 0, 0, 1, 0
  3. Identificadores -> 0, 1, 0, 0
  4. Estilos en linea -> 1, 0, 0,0
  5. !important
    6. El !important rompe la especificidad, es como decirle al navegador, "Quiero que apliques este estilo si o si"

Notas y ejemplos

Si necesitamos hacer uso de varios selectores, que lo mejor es evitarlo debe ir ordenado asi:

  1. Etiquetas
  2. Identificadores
  3. Clases
  4. los selectores por atributos []
          
            blockquote#example.example
Con este selector tendríamos un a especificidad de 0, 1, 1, 1 ya que estamos aplicando el estilo a nuestra etiqueta blockquote que tenga la clase ejemplo y que a su vez tenga una clase ejemplo. Ser tan especifico en un selector es mala practica
!important hará que nuestro estilo rompa la especificidad y cascada, pondrá nuestro estilo a las malas. Lo que no debería pasar ya que es una mala practica

La herencia

Es la capacidad de un selector de obtener (heredar) los valores de sus ancestros mas cercanos, para aplicarla se usa el valor inherit. Si queremos evitarla le podemos asignar otro valor o inicializar nuestra propiedad con initial
No todas las propiedades tienen la capacidad de heredar. Lista de propiedades que si pueden heredar:
