PseudoClases

Daremos estilos dependiendo del contexto, posición o estado del elemento

Las PseudoClases se caracterizan y las podemos usar añadiendo : luego de nuestro selector

        
        .form *:required
En este caso la PseudoClase esta evaluando el estado de nuestro elemento

Esta PseudoClase aplicara estilos a todos nuestros elementos que tengan el atributo required

        
          .form *:valid

valid nos ayudara a aplicar estilos cuando la información que nos haya dado el usuario cumpla con las reglas del input 

        
          .form *:invalid

Este es el contrario que nuestra PseudoClase anterior, aplicara los estilos cuando la información sea invalida o no cumpla con las especificaciones que hemos puesto para nuestro input 

        
          .form *:focus

Cuando nuestro cursor este sobre nuestro elemento. No solo es para los inputs, podríamos usarlo en casi cualquier elemento 

        
          .form *:disabled

Para cuando nuestros elementos tengan el atributo de deshabilitar , que en su mayoría son los inputs

Debemos tener en cuenta cierto orden al aplicar nuestras PseudoClases ya que algunas podrían sobrescribir algunos estilos aplicados anteriormente

El orden recomendado para los enlaces es:

  1. :link
  2. :active
  3. :visited
  4. :hover
        
          .menu a:visited

Esta PseudoClase nos ayuda a identificar los enlaces que ya hemos usado

Si usamos esta PseudoClase luego de nuestro hover, esta la inhabilitara
        
          .menu a:hover

Aplicara nuestros estilos cuando estamos sobre un elemento

Al ser una PseudoClase interactiva, esta debe estar al final de todas.
Se puede aplicar el hover a cualquier elemento
        
          #temario-css:target

Usamos target cuando hemos accedido a los enlaces internos

Por posición

Podemos hacer uso de algunas PseudoClases para acceder a elementos dando referencias
Primer hijo 
        
        .lista li:first-child
Buscara el primer hijo pero que cumpla con la etiqueta que hemos puesto, en este caso sera un elemento li, si el primer hijo no es li. El estilo no se aplicara
Ultimo hijo 
        
        .lista li:last-child
Por posición 
        
        .lista li:nth-child(3)

Accedemos a el hijo por medio de su posición, el primer elemento es el 1

Repetición 
        
          .lista li:nth-child(4n)

La n ayudara a que se repita el estilo según el primer valor, en este caso cada 4 posiciones

Siguiente 
        
        .lista li:nth-child(3n + 1)

En este caso estamos usando nuestro selector del siguiente hermano, por lo tanto haremos que el estilo se le aplique al siguiente elemento del valor que le dimos. Por lo que quedaría igualmente cada 4 elementos, con el detalle que si el 3 elemento era el ultimo el estilo se le aplicara al primero

Pares 
        
        .lista li:nth-child(even)

Con el valor even haremos que a cada 2 elementos se les apliquen nuestros estilos

Impares 
        
        .lista li:nth-child(odd)

A nuestros elementos que estén en una posición impar se les aplicaran nuestros estilos

Por posición y tipo

Recordemos que con first-child podemos acceder a nuestro primer hijo, sin embargo, si este primer hijo no es el elemento que buscamos nuestros estilos no se aplicaran, para este caso, podemos acceder a nuestro elemento por medio de su tipo
Primero en su tipo 
        
          .articulo p:first-of-type

De esta forma buscaremos el primer elemento que sea de tipo párrafo y le aplicaremos el estilo, el estilo no se aplicara solo si no encuentra este elemento

Ultimo en su tipo 
        
          .articulo p:last-of-type

Buscaremos el ultimo párrafo de nuestro articulo

Buscar por tipo 
        
          .articulo p:nth-of-type(3)

Le daremos estilos a nuestro tercer párrafo

También podemos hacer uso del siguiente, impares, pares. Como lo vimos en la parte superior, aunque no es muy usado.

Negación

Excluiremos uno de los elementos de nuestros estilos. Nuestros estilos serán para todos excepto... 
        
          .articulo *:not(p)

Dentro del not definimos cuales elementos(también podemos hacer uso de nuestros selectores) no queremos que se le apliquen nuestros estilos
