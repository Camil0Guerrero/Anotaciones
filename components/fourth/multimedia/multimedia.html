Multimedia

Entendemos por multimedia todo aquello que no es texto

Algunas etiquetas son:

  • img
  • audio
  • video
  • iframe
  • canvas
  • svg
  • picture
  • object
  • embed
  • map
  • area

Para estas etiquetas ademas del width en 100% lo mejor es dejar la altura en auto, para que no se deformen nuestros elementos

En el Responsive Design la multimedia debe ser flexible y adaptarse al tamaño de su contenedor, lo podemos lograr con las siguiente regla CSS: 

      
        img, audio, video, iframe, canvas, svg, picture{
          max-width:100%;
          height: auto;
        }

Para hacer responsivo estos elementos y que tengan siempre la misma resolución tenemos dos opciones, la segunda es mas sencilla e idónea para nuestro tiempo

Imágenes Responsivas

Simplemente adaptamos la imagen al tamaño de su contenedor.

Atributo srcset y sizes

El propósito de estos atributos, es mejorar el rendimiento de carga, al colocar la misma imagen pero en diferentes tamaños y pesos, dependiendo del tamaño o la resolución de la pantalla.

No todos los navegadores, sobretodo los antiguos soportan estos atributos. Por lo tanto es recomendado definir en el src la imagen mas pequeña por si hay un error

Este atributo lo podemos manejar con la resolución, hay dispositivos que manejan distintas resoluciones y nuestro navegador lo tomara asi, para definir distintos tamaños y la resolución de cuando se van a manejar es necesario definir la ruta de nuestra imagen y luego declarar la resolución de cuando queremos que cargue. En cuanto hayamos definido estos 2 valores separamos con una coma (,) y podemos declarar el siguiente

Podemos mirar este comportamiento usando las devtools y simular un Iphone o aumentar el zoom de nuestro navegador, el 150% seria el 1.5x por ejemplo. 

      
        <img
        src="img/html5-3d-600.png"
        srcset="img/html5-3d-600.png 1x, img/html5-3d-900.png 1.5x, img/html5-3d-1200.png 2x"
            alt="Logo de Html en 3D"/ 
        >

La otra manera de manejar cuando se va a cargar una imagen u otra es por definiendo en pixeles el máximo de cada una. En lugar de px ponemos w refiriéndonos al width, ancho

En este caso estamos definiendo la primera imagen para cuando haya menos de 600px, nuestra segunda resolución se hara cuando estemos entre los 600 y 900px,la ultima resolución se aplicara cuando estemos a mas de 1200px 

      
      <img
      src="img/html5-3d-600.png"
      srcset="img/html5-3d-600.png 600w, img/html5-3d-900.png 900w, img/html5-3d-1200.png 1200w"
      alt="Logo de Html en 3D"
      />

Adicionalmente tenemos el atributo sizes, en el cual podemos manejar como si fuesen media queries, podemos definir propiedades y luego lo que queremos que haga mientras estas se cumplen

Si queremos ver ejemplos podemos ver las paginas hechas en WordPress, ya que estos usan mucho estos atributos. Cuando subimos una imagen a su servidor WordPress se encarga de crear varias imágenes con distintas resoluciones para precisamente luego poder hacer que nuestras imágenes siempre se vean lo mejor posible

Etiqueta picture

El objetivo de esta etiqueta, no es sólo mostrar diferentes tamaños de la misma imagen, sino también mostrar imágenes diferentes. Por ejemplo, recortar una imagen de forma diferente según el tamaño de la pantalla y las diferencias en el diseño. Esto se conoce como "dirección de arte".

El orden recomendado es de mayor a menor

A diferencia de nuestras imágenes normales donde poníamos el src para navegadores antiguos. En picture definimos una imagen con la mas pequeña. Esta imagen se aplicara cuando nuestra pantalla sea menor que 600px


webp -> Es un tipo de formato propuesto por Google, este trata de ser una mejor solución de imagen que los jpg y los gif animados con un mayor peso
Con picture no solo podemos evaluar y definir las media queries, también podemos definir un formato de imagen poco usado y aplicarlo solo si nuestro navegador lo soporta

Videos Responsivos

Para los videos tenemos dos formas de hacerlos responsivos y que no pierdan su resolución a medida que hay menos espacios, estas formas son:

Técnica vieja

      
      .responsive-media {
        position: relative;
        max-width: 100;
        height: 0;
        padding-bottom: 56.25%;
      }

Con el padding haremos el efecto de tener una altura, como si empujáramos nuestro video hacia la parte de arriba

El valor del padding se debe al formato widescreen 16/9, que es el recomendado usar en los videos e imágenes.

Este valor lo obtenemos con una regla de 3

      
        16  -> ancho 100%
        9 -> 56.25%

Nueva forma

Con la propiedad aspect-ratio definimos las proporciones que queremos manejar, como primer valor definimos el alto y de segundas el ancho. 

      
      .aspect-ratio-16-9 {
        background-color: rebeccapurple;aspect-ratio: 16 / 9;
      }

Iframes Responsivos

El problema de aplicar a los iframe estilos generales para hacerlo responsivo, es decir width: 100% y height: auto; vamos a tener un problema y es que nuestro navegador va a tomar 150px por defecto para nuestra altura, por lo que no se vera muy bien

Con nuestra vieja técnica es una muy buena manera de hacer que se vea bien siempre

Con la nueva técnica tendremos el mismo efecto tan solo que esta forma si respetara el tamaño que le hemos definido a nuestro elemento

Para los iframe también podemos definir como queremos manejar la resolución, esto nos ayuda a que al hacer a nuestro elemento responsive no se nos deforme 

      
        aspect-ratio: 1/1;
