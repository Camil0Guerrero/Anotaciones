Procesadores

Preprocesadores

Son herramientas que toman un lenguaje y lo transforman en CSS

lo encontramos en los distintos frameworks y en grandes compañías ya que permite trabajar a varias personas aplicando estilos y luego agruparlos dentro de una sola hoja

Beneficios:

  • Nos permiten tener mejor organizado nuestro código
  • Estos tienen mas funcionalidades al momento de escribir el código CSS

Estos pre procesadores hacen uso de Nodejs para la compilación

Stylus

No necesitamos agregar las llaves luego de nuestro selector este lo maneja con indentación

Tampoco necesitamos puntos y comas;

Podemos crear mixins -> Es como si fuera una función

Con la directiva @import mandamos llamar otros archivos

Podemos crear ciclos

Less

CSS con algunas cosas mas

Nesting -> Anidación dentro de CSS, dentro de un mismo selector poder definir los estilos para todos sus elementos internos, ademas, definir PseudoClases y PseudoElementos

Sass

CSS con super poderes

Es creado con Ruby

Es casi el estandar dentro de los preprocesadores

Estos archivos pueden tener como extension sass o scss -> "super css" que es la mas convencional

Definimos las variables con $

Post procesadores

Son herramientas que procesan el CSS al finalizar, lo optimizan y lo automatizan

Pueden agregar el soporte para las propiedades nuevas

minifican la hoja de estilos

Post CSS

Es el mas popular

Trabaja con JavaScript

CSSnext

Nos da la posibilidad de usar las características antes de que se agreguen a CSS

CSS in JS

En nuestro tiempo las librerías de JavaScript están en su auge y los desarrolladores tienen la posibilidad de definir los estilos dentro de sus propios componentes

Estos son recomendados usar sobretodo cuando estamos en grandes proyectos, cuando nos estamos encargando de tan solo una parte del código y lo necesitan rápidamente o en ese formato

Herramientas

  • codepen -> Herramienta para practicar
  • Autoprefixer -> Nos dará el soporte para todos los navegadores
  • Minifier -> Nos dará nuestro código minificado
