Frameworks CSS

Son marcos de trabajo que nos ofrecen componentes y utilidades de UI

Menos relevantes en esta época

960 Grid system

Salio en la época cuando no había responsive design y donde la pantalla estandar media 1024px y aplicaba 32px de margen a cada lado. Por ello se llama asi

Skeleton

Pure CSS

Era un framework orientado a componentes, podemos descargar solo lo que necesitamos

Ink

Te permite configurar el código, los nuevos frameworks nos dan el código con la posibilidad de manipularlo a nuestras necesidades, lo podemos obtener con css puro o con el preprocesador Sass

Aunque no es recomendado manipular el código fuente ya que en cuanto el framework se actualice nuestros cambios no se verán, por lo que lo recomendado es que agreguemos una clase con nuestro código en css puro

Muicss

Podemos implementarlo muy fácil con librerías de JavaScript como react, angular, etc...

Semantic UI

Utiliza de una manera mas amigable el uso de todas las etiquetas semánticas que tenemos en HTML

Tiene un propio estilo visual, no tiene los mismos estilos decadente de bootstrap y foundation

Los mas usados o relevantes

Estos nuevos frameworks ya tienen accesibilidad web

BULMA

No nos da el JavaScript, en algunos componentes nos da la recomendación o sugerencia de como lo podemos hacer

Fue desarrollado por los mismos creadores de vue.js por lo que es común ver que se hagan publicidad entre ellos

Uikit

Parecido a MUI por su facilidad de implementación en las librerías de JavaScript

La mayoría de sus clases inician con uk-

Materialize

Es muy reconocido por haber sido el primer framework que dejo de usar jquery y empezó a usar JavaScript

Lo sobresaliente son los diseños de sus componentes

Foundation

Es el segundo framework mas reconocido, se ha mantenido junto a Bootstrap creciendo y adaptándose a las nuevas tecnologías, tal vez sobresale un poco en las buenas practicas que maneja en su código y que todos los componentes ademas de su documentación tiene video tutoriales

Tailwind CSS

Ha ganado mucha popularidad porque es muy fácil de implementar, tiene muchas clases utilitarias y bastantes componentes. Al igual nos permite eliminar cosas que no vamos a usar en nuestro proyecto, al igual es muy fácil agregarlo al flujo de trabajo con react y todas estas librerías e incluso tiene su documentación

Bootstrap
