Transition

Las transiciones nos ayudan a que en lugar de que los cambios en nuestras propiedades se hagan de forma instantánea o de golpe, definir una animación o un intervalo de tiempo para que estos cambios se hagan

La propiedad transition es la que usamos para definir estos estilos, aunque es el shorthand, este si es muy funcional y recomendado.

Los intervalos es una parte importante de entender ya que es como nuestra animación se va a comportar en el tiempo, esto lo podemos definir con distintas formas:

Intervalos de transición

  • Ease -> Inicia lentamente, en el medio acelera y finaliza lento. Por defecto
  • Ease in -> Inicia lento y va acelerando gradualmente hasta el final
  • Ease-out -> Inicia rápidamente y desacelera gradualmente hasta el final
  • Ease-in-out -> Inicia lentamente, acelera en el medio y finaliza lento, parecido a ease pero este es mas pronunciado
  • Linear -> Tendrá una velocidad constante
  • step-start -> La transition saltara instantáneamente al final
  • step-end -> Estará en el estado inicial hasta que al final hara un salto subito
  • steps -> Debemos poner entre parentesis un numero natural, lo cual hara que nuestra animación se haga en los pasos que le hayamos definido

Propiedades

El shorthand es la mejor opción para definir nuestras transiciones, sin embargo, es indispensable saber que estamos definiendo y saber que otras opciones tenemos.

Ordenado como debemos definirías en el shorthand
  1. transition-property -> Propiedad a la que queremos aplicar la animación
  2. transition-duration -> Duración de nuestra transición. Lo podemos definir con milisegundos (ms) y segundos (s)
  3. transition-timing-function -> Definimos nuestra función de intervalo de transición, las cuales son los anteriores ejemplos
  4. transition-delay -> Tiempo de espera, antes de nuestra animación. se puede con ms o s

Notas:

  • El transition se aplicara apenas recarguemos o se cargue nuestra pagina, por lo que las propiedades que definimos desde el inicio estarán con la transición
  • Debemos tener en cuenta lo que definimos ya que no es lo mismo usar el shorthand a usar la propiedad especifica
  • all Nos ayuda a definir el transition para todas las propiedades
  • A muchas personas no les gusta ya que puede haber problemas con el rendimiento, sin embargo, esto es cuando tenemos muchas animaciones, cuando son varias o tenemos varias transiciones es mejor ser específicos
  • La mala practica es aplicar varias animaciones a un mismo elemento en un mismo estado
  • Si quisiéramos aplicar una transición distinta a cada propiedad no podríamos usar all
  • Al usar all, se les aplicara la animación a todas las propiedades, incluso a las que definimos al inicio. Entonces es mejor tener el control de todas las propiedades animadas
  • Hay propiedades que no se pueden animar. Propiedades que si se pueden animar
  • Si queremos definir transition a varias propiedades específicamente, podemos hacerlo separando con comas (,). Por Ejemplo: transition: opacity 1s ease-out, border-color 3s steps(3), filter 2s ease-in 1s, box-shadow 1.5s linear 2s;

Transformaciones 2D

Podemos hacer uso de funciones para hacer que nuestros elementos tengan movimientos y hagan ciertas cosas. Estas funciones las debemos definir usando la propiedad "transform"

Translate

Nos ayudara a mover nuestro elemento, este tomara como referencia su tamaño, tenemos las propiedades especificas: translateY y translateX, en este caso usaremos su shorthand translate en la cual debemos pasar como primer valor el movimiento en X y de segundo el del eje Y. estos toman como referencia left y top por lo que al dar un valor positivo al eje X ira hacia la derecha y al darle un valor positivo en Y ira hacia abajo, si les asignamos un valor negativo ira al lado inverso, es decir, para el eje X, izquierda y para el eje Y, hacia arriba.

Rotate

Podemos definir la rotación en grados y radianes, aunque lo mas usado son los grados ya que es lo mas sencillo

Igualmente tenemos la rotación para el eje X y Y, sin embargo estos se notan mejor con perspectiva

El que podemos usar es el rotateZ, este efecto se aplica hacia la dirección de las manecillas del reloj, es decir, de izquierda a derecha

Skew

Esta transformación es unica del 2d, es decir, sin perspectiva. Lo que podemos hacer con ella es inclinar nuestro elemento, es como si tomáramos uno de los extremos y lo empujáramos. Esta función recibe la inclinación en grados

Si le asignamos un valor positivo a skewX la inclinación ira hacia la izquierda

Para skewY el valor positivo sera hacia arriba, hacia top

Con el shorthand podemos definir ambos, de primeras el eje X y como segundo parámetro la inclinación del eje Y.

Transformaciones 3D

Para manejar la ilusión óptica de las 3 dimensiones debemos usar la perspectiva, esta recibe un valor numérico con cualquier unidad de medida, este va a definir la profundidad que vamos a manejar

La distancia que definimos digamos que es la distancia entra la pantalla y nuestro elemento, por lo tanto, entre menos sea, mas veremos el efecto

La perspectiva la podemos definir en cada una de nuestras propiedades que las necesiten o la podríamos definir en el elemento padre

En el elemento padre la podemos definir con la propiedad perspective o en cada propiedad, por ejemplo: transform: perspective(10rem) translateZ(-4rem);

Como hemos agregado la perspectiva a nuestros elementos, tenemos un nuevo eje, el eje Z, para manejar este eje todas nuestras funciones tienen su propia función, y los shorthands para estas 3 funciones de cada elemento tendrán el mismo nombre solo que añadido el "3D"

Estas propiedades ya las hemos manejado en el 2D, a continuación veremos algunos ejemplos de lo que podemos hacer con nuestros 3 ejes

  1. transform: translate3d(2rem, 50%, 5rem);
  2. transform: scale3d(2, 2, 3);

    3. Cuando ya tenemos perspectiva podemos ver como nuestro elemento se acuesta y va rotado sobre su eje x

  3. transform: rotateX(60deg);
  4. transform: rotateX(90deg);

    5. Igualmente, es como si pusiéramos un punto fijo en la parte del centro arriba y otro en el centro abajo y luego empujáramos nuestro elemento.

  5. transform: rotateY(60deg);
  6. transform: rotateY(90deg);

Origin

En los ejemplos anteriores todos nuestros elementos estaban aplicando los estilos a partir del centro ya que este es el valor por defecto. Esto no significa que no podemos cambiarlo, para ello tenemos la propiedad "transform-origin"

Esta propiedad recibe como parámetro los valores de los ejes: X Y y Z, estos los podemos definir con cualquier medida aunque se recomienda con porcentajes (Tomara como referencia la esquina de arriba a la izquierda), igualmente podemos usar las palabras clave

Cuando aplicamos un solo valor al otro se le aplicara por defecto center

  1. top left : 0 0
  2. top right : 100% 0
  3. bottom right: 100% 100%
  4. bottom left : 0 100%
